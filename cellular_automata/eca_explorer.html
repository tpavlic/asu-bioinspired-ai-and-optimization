Reading the rule table: each column shows one 3-cell neighborhood (top row, green = 1) and the new center cell it produces (bottom, amber = 1). Columns run from 111 (index 7) on the left down to 000 (index 0) on the right. The rule number is the 8-bit integer whose bits are exactly the output row read left-to-right (bit 7 → bit 0). Viewing the rule in hex lets you immediately see which output bits are set without counting. The slider, number display, and lookup table all stay synchronized — you can drive the rule from any of them.
